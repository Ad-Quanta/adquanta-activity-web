# 业务逻辑迁移到原生接口指南

## 概述

为了遵循"尽量在 SDK 中处理，减少 Web 端工作"的设计原则，我们将业务逻辑从 `activity-runtime-sdk.js` 迁移到了原生 SDK 接口。

## 迁移前后对比

### 迁移前（需要手动引入 JS SDK）

```javascript
// 需要手动 import
import { ActivityRuntimeSDK } from "./activity-runtime-sdk.js";

// 创建实例
const sdk = new ActivityRuntimeSDK({
  activityId,
  code,
  token,
  channelTag,
  apiBaseUrl: "",
});

// 初始化
await sdk.init();

// 追踪事件
await sdk.trackEvent("ad_watch_completed", {...});
```

### 迁移后（使用原生接口，零集成）

```javascript
// 无需 import，原生 SDK 自动注入

// 初始化活动会话
const session = await ActivityBridgeHelper.initActivity(activityId, code, token, channelTag);

// 追踪事件
await ActivityBridgeHelper.trackEvent("ad_watch_completed", {...});

// 获取会话信息
const session = ActivityBridgeHelper.getSession();
```

## 可用的原生接口

### 1. 初始化活动会话

```javascript
// 方法签名
ActivityBridgeHelper.initActivity(activityId, code, token, channelTag)

// 示例
const session = await ActivityBridgeHelper.initActivity(
  "welfare_center_202512",
  "abc123",
  "",
  "app"
);

// 返回格式
{
  "userId": "user_123456",
  "deviceId": "dev_1234567890",
  "activityId": "welfare_center_202512",
  "channelTag": "app",
  "sessionToken": "sess_1234567890",
  "issuedAt": "Mon Jan 08 12:00:00 CST 2024",
  "expiresInSeconds": 3600
}
```

### 2. 追踪事件

```javascript
// 方法签名
ActivityBridgeHelper.trackEvent(eventType, eventData)

// 示例
await ActivityBridgeHelper.trackEvent("ad_watch_completed", {
  taskId: "task_watch_ad",
  reward: 196,
  success: true
});

// 返回格式
{
  "accepted": true,
  "event": {
    "eventId": "evt_1234567890",
    "activityId": "welfare_center_202512",
    "userId": "user_123456",
    "eventType": "ad_watch_completed",
    "eventData": {...}
  }
}
```

### 3. 获取会话信息

```javascript
// 方法签名
ActivityBridgeHelper.getSession()

// 示例
const session = ActivityBridgeHelper.getSession();

// 返回格式（与 initActivity 相同）
{
  "userId": "user_123456",
  "deviceId": "dev_1234567890",
  ...
}
```

### 4. 抽奖功能（可选，待实现）

```javascript
// 方法签名
ActivityBridgeHelper.drawLottery(activityId)

// 示例
const result = await ActivityBridgeHelper.drawLottery("welfare_center_202512");
```

### 5. 领取奖励（可选，待实现）

```javascript
// 方法签名

// 示例

```

## 迁移步骤

### 步骤 1：移除 ActivityRuntimeSDK 的 import

```javascript
// 删除这行
// import { ActivityRuntimeSDK } from "./activity-runtime-sdk.js";
```

### 步骤 2：替换初始化代码

**迁移前：**
```javascript
const sdk = new ActivityRuntimeSDK({
  activityId,
  code,
  token,
  channelTag,
  apiBaseUrl: "",
});

const session = await sdk.init();
```

**迁移后：**
```javascript
const session = await ActivityBridgeHelper.initActivity(activityId, code, token, channelTag);
```

### 步骤 3：替换事件追踪代码

**迁移前：**
```javascript
await sdk.trackEvent("ad_watch_completed", {
  taskId: "task_watch_ad",
  reward: 196
});
```

**迁移后：**
```javascript
await ActivityBridgeHelper.trackEvent("ad_watch_completed", {
  taskId: "task_watch_ad",
  reward: 196
});
```

### 步骤 4：替换获取会话代码

**迁移前：**
```javascript
const session = sdk.getSession();
```

**迁移后：**
```javascript
const session = ActivityBridgeHelper.getSession();
```

## 注意事项

1. **异步方法**：`initActivity` 和 `trackEvent` 返回 Promise，需要使用 `await` 或 `.then()`
2. **错误处理**：建议使用 `try-catch` 包裹异步调用
3. **SDK 加载**：确保在调用前等待 SDK 加载完成（原生会自动注入）
4. **向后兼容**：如果原生 SDK 未注入，Web SDK 辅助库会提供降级方案

## 优势

✅ **零集成工作**：Web 端无需引入任何 JS 文件  
✅ **统一管理**：业务逻辑在原生端统一处理  
✅ **易于维护**：业务变更只需更新原生 SDK  
✅ **性能更好**：减少 JS 执行，逻辑在原生端处理  
✅ **设计一致**：符合"尽量在 SDK 中处理"的原则
